# Instruction-Tuning Dataset — Week 3 Deliverable (D2)

Dataset for fine-tuning a Small Language Model (Qwen 2.5 3B Instruct)
to perform offline log analysis, as specified in the internship
document Section 5.3 (Dataset Creation) and Section 8 (Deliverable D2).

## Files

| File | Examples | Purpose |
|---|---|---|
| `logs_dataset.jsonl` | 1,940 | Complete dataset (all examples) |
| `train.jsonl` | 1,550 (79.9%) | Fine-tuning training set |
| `val.jsonl` | 192 (9.9%) | Validation set (per-epoch evaluation) |
| `test.jsonl` | 198 (10.2%) | Held-out test set (Week 4 baseline + Week 6/8 evaluation) |

The split is **80/10/10, stratified by severity** (seed = 42, reproducible).
There is no example overlap between splits (verified: zero leakage).

## Format

Each line is one JSON object with three fields (instruction-tuning format):

```json
{
  "instruction": "<system prompt from document Section 5.3>",
  "input": "<normalized log window>",
  "output": "SEVERITY: ...\nINCIDENT_TYPE: ...\nROOT_CAUSE: ...\nSUMMARY: ...\nRECOMMENDED_ACTIONS:\n1. ...\n2. ..."
}
```

The output always contains the 5 required fields:
`SEVERITY` (CRITICAL | HIGH | MEDIUM | LOW | INFO), `INCIDENT_TYPE`,
`ROOT_CAUSE`, `SUMMARY` (2-3 sentences), `RECOMMENDED_ACTIONS` (numbered list).

## Sources

Per document Section 5.3: *"Sources: open-source log datasets (LogHub,
Loghub-2.0), synthetic generation using templates, and manually curated
examples."*

### 1. LogHub (1,865 examples after deduplication)

Log windows extracted by the Week 2 preprocessing pipeline
(`src/preprocessing/preprocessor.py`) from 16 LogHub sources
(2,000 lines sampled per source), stored in `data/processed/`:

Android, Apache, BGL, HDFS, HPC, Hadoop, HealthApp, Linux, Mac,
OpenSSH, OpenStack, Proxifier, Spark, Thunderbird, Windows, Zookeeper.

Labels are produced by a **source-aware rule-based label generator**
(`src/training/label_generator.py`): each source has a dedicated rule
set matching its known log patterns (e.g. SSH brute-force sequences in
OpenSSH, fatal TLB hardware errors in BGL, quorum connection breaks in
Zookeeper), with a generic severity-keyword fallback.

### 2. Synthetic templates (75 examples)

Generated by `src/training/synthetic_generator.py` from 33
manually-curated, parameterized incident templates covering:
database (pool exhaustion, deadlock, replication lag, slow queries),
memory (OOM kill, JVM heap exhaustion), disk (space exhaustion,
I/O failure), network (DNS failure, connection refused, TLS expiry,
packet loss), security (brute force, lockout, privilege escalation,
port scan), application failures (unhandled exceptions, segfaults,
config errors), infrastructure (health-check failure, crash loops,
LB degradation, cascading timeouts, queue backlog), performance
(latency regression, CPU saturation), and routine/benign events
(deployments, backups, autoscaling, rate limiting) so the model
learns non-incident classification (supports the < 10% false-positive
target, document Section 6.1).

Input logs and output labels are rendered from the same variant
values, guaranteeing label/input consistency. All placeholder tokens
(`<TIMESTAMP>`, `<IP_ADDR>`, `<PORT>`, `<USER_ID>`, `<FILE_PATH>`,
`<MEM_ADDR>`, `<UUID>`) match the Week 2 normalization vocabulary.

## Statistics

**Severity distribution (full dataset):**

| Severity | Count | Share |
|---|---|---|
| INFO | 986 | 50.8% |
| HIGH | 462 | 23.8% |
| LOW | 247 | 12.7% |
| MEDIUM | 200 | 10.3% |
| CRITICAL | 45 | 2.3% |

**Incident types:** 79 distinct categories; largest class
("Routine System Activity") is 16.9% of the dataset.

**Quality checks (all passing):**
- 1,940 examples (document requires 1,000+)
- Valid JSONL, uniform `{instruction, input, output}` schema
- All outputs contain the 5 required fields with valid severity values
- No placeholder/vague text
- No duplicate inputs; zero overlap between train/val/test
- Single consistent instruction (system prompt from document Section 5.3)

## Regeneration

From the repository root (fully offline, standard library only for the split):

```bash
python -m src.training.build_dataset   # -> logs_dataset.jsonl
python -m src.training.split_dataset   # -> train/val/test.jsonl
```
